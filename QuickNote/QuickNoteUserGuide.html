Note Colors

QuickNote supports both automatic category-based colors and custom note colors.

Automatic Category Color (New! ‚ú®)

  1. Default Behavior
    • New notes automatically inherit their category's color
    • Each category has a unique pastel color
    • Makes notes visually organized by category
  2. How It Works
    • Create a note in Personal category ‚Üí Gets blue color
    • Create a note in Work category ‚Üí Gets green color

Custom Note Color

  1. Set Custom Color
    • Open a note in the editor
    • Tap the Color Palette icon in the app bar
    • Select from predefined colors
    • The note card will display in your chosen color
    • Custom color overrides category color
  2. Reset to Category Color

  1. Edit Category
    • Go to Manage Categories
    • Tap on a category to edit
    • Choose a new color from the palette
  2. Automatic Note Updates
    • All notes using that category color will update automatically
    • Notes with custom colors remain unchanged
    • Only affects notes that haven't been manually colored

üíæ Backup and Restore

‚ö†Ô∏è Important: Always backup before major updates or device changes!

Understanding Backup Files

  • Format: JSON file with .backup extension
  • Contents: All notes, categories, and metadata
  • Schema Version: Tracks database version (currently v6)
  • Compatibility: Backward compatible with older backups

Creating a Backup

Method 1: Save to Device

  1. Open Backup Screen: Tap Menu ‚Üí Backup & Restore
  2. Create Backup: Tap Backup ‚Üí Save notes to device storage
  3. Choose Location: Native file picker opens. Navigate to desired folder. Filename suggestion: quicknote.backup
  4. Confirmation: Green snackbar: "Backup saved successfully!"

Method 2: Share Backup

  1. Open Backup Screen: Tap Menu ‚Üí Backup & Restore
  2. Share Backup: Tap Share Backup ‚Üí Share backup file via email or cloud
  3. Choose App: Share sheet appears with installed apps (Email, Google Drive, Dropbox, etc.)
  4. Send/Upload: Complete the sharing action in chosen app

Restoring from Backup

Method 1: Restore (Merge)

Use When: You want to add backup data to existing notes (no deletions)

  1. Open Backup Screen: Tap Menu ‚Üí Backup & Restore
  2. Start Restore: Tap Restore (merge) ‚Üí Add backup data to existing notes
  3. Select Backup File: File picker opens. Browse to your .backup file
  4. Import Process: Notes and categories import. Duplicates: Existing IDs are skipped. New items: Added to database
  5. Completion: Snackbar: "Restore completed (merge)." Status shows: "Restored: X notes, Y categories"

Method 2: Restore (Replace)

‚ö†Ô∏è Warning: This will delete all existing notes and categories before importing!

Use When: You want to completely replace all data with backup (destructive!)

  1. Open Backup Screen
  2. Start Replace: Tap Restore (replace) ‚Üí Replace all notes with backup data
  3. Confirm Action: Warning dialog appears. Tap Yes, replace to proceed
  4. Select Backup File: Choose .backup file
  5. Import Process: All existing notes deleted, all categories deleted (except defaults), backup data imported fresh

Backup Best Practices

  1. Regular Backups: Create weekly backups (or before major changes). Store in multiple locations (device + cloud)
  2. Before Updating: Always backup before app updates
  3. Test Restores: Occasionally test restore on a different device
  4. Naming Convention: Use descriptive names: quicknote_2025-10-04.backup
  5. Cloud Storage: Upload to Google Drive, Dropbox, or OneDrive

Backup Compatibility

Backup Version Current App Result
v3, v4, v5 (older) v6 (current) ‚úÖ Works perfectly (plain text)
v6 (current) v6 (current) ‚úÖ Perfect match (with Rich Text)
v7+ (future) v6 (current) ‚ö†Ô∏è Warning shown, user decides

üì§ Sharing Notes

Share a Single Note

  1. Open the Note: Tap the note in the list to open editor
  2. Tap Share Icon: Look for Share (üì§) icon in app bar
  3. Choose Share Option:
    • Share full note: Includes both title and content
    • Share content only: Shares content without title (disabled if note has no content)
  4. Select App: System share sheet appears (Email, WhatsApp, SMS, etc.)
  5. Shared Content: Rich text formatting converts to plain text. Images are referenced by path
Option What's Shared When to Use
Share full note Title + Content Sending complete note with context
Share content only Content only (no title) Quick sharing without title overhead

Share Multiple Notes (Workaround)

QuickNote doesn't have bulk share, but you can:

  1. Use Backup: Create a backup file, share the .backup file. Recipient can restore on their device
  2. Copy-Paste: Open each note, copy content manually, paste into email/document

üîß Troubleshooting

App Performance Issues

Problem: App feels slow or laggy

Solution:

  1. Clear app cache (Settings ‚Üí Storage ‚Üí Clear Cache)
  2. Reduce number of large images in notes
  3. Delete very old unused notes
  4. Restart your device

Note Editing Issues

Problem: Rich text formatting not working

Solution:

  1. Ensure toolbar is visible (tap Aa icon)
  2. Select text before applying formatting
  3. Try restarting the app

Backup/Restore Errors

Problem: "Invalid backup format" error

Solution:

  1. Verify file has .backup extension
  2. Don't manually edit backup files
  3. File may be corrupted; try another backup

Search Issues

Problem: Search returns no results

Solution:

  1. Check spelling of search terms
  2. Try partial words (e.g., "doc" instead of "document")
  3. Ensure notes exist in database

üîí Privacy and Security

Data Storage

  • Local Storage: All notes stored in SQLite database on your device
  • Location: Internal app storage (not accessible to other apps)
  • Encryption: Data stored in plaintext (device security applies)

Data Collection

  • Personal Data: QuickNote does NOT collect personal information
  • Analytics: No tracking or analytics (fully offline app)
  • Ads: No advertisements or ad networks

Best Practices

  1. Lock Your Device: Use PIN/pattern/fingerprint on your phone
  2. Secure Backups: Store in password-protected cloud storage
  3. Be Careful Sharing: Don't share sensitive notes publicly
  4. Regular Backups: Protect against device loss/damage
  5. App Updates: Keep QuickNote updated for security patches

üí° Tips and Tricks

Productivity Tips

  2. Favorite System: Star active projects or current tasks. Unstar when complete. Quick filter to see what's active

  4. Selective Sharing: Share full note when context is important. Share content only for quick information transfer

‚ùì Frequently Asked Questions (FAQ)

Q: Is QuickNote free?

A: Yes! QuickNote is completely free with no ads or in-app purchases.

Q: Does QuickNote work offline?

A: Yes! All features work without internet. Only sharing via cloud apps requires internet.

Q: Can I sync notes between devices?

A: Not automatically. Use backup/restore to manually transfer notes between devices.

Q: Can I add images to notes?

A: Yes! Use the image button in the rich text toolbar. Tap inserted images to view full-screen with zoom.

Q: Can I zoom into images in notes?

A: Yes! Tap any image thumbnail in a note to open full-screen viewer. Use pinch-to-zoom and pan gestures to explore the image in detail.

Q: Why does my note automatically have a color?

A: New notes inherit their category's color automatically. You can override this by applying a custom color via the palette icon.

Q: How do I change all notes in a category to a new color?

A: Edit the category color in Manage Categories. All notes using that category's default color will update automatically (notes with custom colors won't change).

Q: What's the difference between "Share full note" and "Share content only"?

A: "Share full note" includes both title and content, while "Share content only" shares just the note body without the title. Choose based on whether recipients need context from the title.

Q: How do I recover deleted notes?

A: Use your most recent backup file with Restore (merge). No built-in trash bin currently.

Q: How do I request features or report bugs?

A: Use Menu ‚Üí Send Feedback in the app to contact developers.
